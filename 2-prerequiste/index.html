Preparation steps

If you have access to the AWS Management Console with administrative privileges, you can use your AWS account to initiate this workshop.

  1. AWS CloudFormation template: Download
  2. Sample audio file: Download
  3. Source Code: Download

Launch a CloudFormartion Stack

  • Navigate to the AWS Management Console
  • Select the region: us-east-1 (N. Virginia)
  • Search for “CloudFormation” in the AWS Console.
  1. Click Create stack and select With new resources (standard).

  1. In the Create Stack wizard:
    • Select Upload a template file.
    • In the Specify Template section :
      • Select Upload Template File.
      • Click Choose file .
      • Navigate to the folder where you downloaded the AWS CloudFormation file and select the main.yml file.

  1. Specify Stack Details:
  • Enter the stack name (e.g., ai-lab).
  • Keep the default values for the parameters.
  • Click Next.

  1. Click Next in the “Configure stack options” section.
  • Review the configuration in the “Review and create” section. Scroll to the bottom, check the acknowledgment box.
  • Select the Submit button to create the resource.

After completing CloudFormation creation:

  • S3 buckets:
    • sample-data-xx: Stores audio files for processing.
    • hs-output-xx: Stores transcription results from HealthScribe.

  • SageMaker Notebook named SageMakerNotebookInstance-xx in us-east-1 .
  • Source Code Download and import it into the SageMaker notebook instance.
  • IAM Role to execute Jupyter Notebook operations.

Uploading the Audio File to S3

  • Navigate to Amazon S in the AWS Management Console.
  • Select the bucket that starts with sample-data-xx.

  • Click Upload > Add Files.
  • Select the previously downloaded audio file.

  • Click Upload at the bottom of the page to complete the upload.

Updating the IAM Role

  • Navigate to IAM in the AWS Management Console.
  • Click Roles on the left menu.
  • Search for your stack name (ai-lab).
  • Select the role that contains »-SageMakerNotebookRole-XX.
  • Click Add Permissions > Create Inline Policy.

  • In Select Service, search and select IAM.

  • In Allowed Actions, enter PassRole.
  • Check the box next to PassRole.
  • Select All resources.

  • Click Next.
  • Enter a name for the policy.

  • Create Policy to finalize.

Once done. Find and save the following information:

  • [S3_BUCKET_NAME] : The S3 URI where the audio file was uploaded.
  • [OBJECT_NAME] : The full file name, including the extension (e.g., lower-back-consultation.m4a).
  • [IAM_ROLE] : The ARN of the IAM role (»-SageMakerNotebookRole-XX).
  • Example:
    • os.environ[’S3_BUCKET_NAME’] = ‘sample-data-xxxx’
    • os.environ[’OBJECT_NAME’] = ’lower-back-consultation.mp4'
    • os.environ[’IAM_ROLE’] = ‘arn:aws:iam::XXXX:role/XXXX-SageMakerNotebookRole-XXXX’

Enable Amazon Bedrock Model Access:

  1. Access to Amazon Bedrock models is not granted by default. To enable access:
    • Switch to the Oregon (us-west-2) region.
    • Select Model Access from the left-hand menu in the Amazon Bedrock console.
    • Click Enable Specific Models.

  1. Check the boxes next to the models you want to enable: Claude 3 Sonnet and Claude 3 Haiku (or the latest version available, e.g., Claude 3.5).

  1. Click Next to preview the changes.

  1. Review the changes and click Submit. Changes may take a few minutes to apply.
  2. If successful, the Access Status will change to Granted.

Opening Jupyter Notebook in Amazon SageMaker:

  1. Open the Amazon SageMaker console in AWS us-east-1 (N. Virginia).

  1. Click Notebooks in the left menu to list available instances.

  1. Locate the notebook instance starting with SageMakerNotebookInstance- and click Open JupyterLab.
